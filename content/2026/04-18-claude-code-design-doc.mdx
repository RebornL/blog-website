## 准备阶段

### 创建 CLAUDE.md 指引 Claude

```markdown
# 项目概述
这是一个 [项目名称]，目标 [一句话描述]。

## 文档规范
- 使用中文输出
- 设计文档采用：概述、架构设计、模块设计、数据模型、接口设计、风险评估
- 架构图用 Mermaid 语法
```

## 推荐对话流程

1. **需求分析**：让 Claude 阅读需求文档，总结核心需求、功能点、非功能需求、潜在风险
2. **架构设计**：设计系统架构（Mermaid）、技术选型、模块划分、数据流
3. **详细模块设计**：模块职责、类/函数设计、数据结构、依赖关系
4. **接口设计**：请求/响应格式、错误码规范、鉴权机制
5. **数据模型设计**：ER 图、表结构、索引策略
6. **评审与迭代**：检查设计文档与需求文档的一致性

## 一次完成的 Prompt 模板

```
我的需求文档在 docs/requirements.md，请基于这份需求，
输出一份完整的设计文档，保存在 docs/design/architecture.md。

要求：
- 使用中文
- 包含：系统概述、架构设计、模块设计、数据模型、API接口设计
- 架构图用 Mermaid 语法
- 每个模块说明输入输出
- 最后列出设计中需要进一步确认的问题
```

## 技巧

| 技巧 | 说明 |
| --- | --- |
| 先分析再设计 | 让 Claude 先总结需求，确认理解正确后再开始设计 |
| 分模块生成 | 大系统不要一次生成，分模块逐步输出 |
| 要求 Mermaid 图 | Claude Code 支持直接渲染 |
| 迭代修正 | 生成后说"第三部分太简略，请补充" |
| 指定格式 | 如果有内部模板直接传给 Claude |
