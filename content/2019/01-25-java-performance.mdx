## 性能分析方法

- **自顶向下**：从应用层向下寻找优化机会。监控操作系统资源、JVM、Java EE 容器及应用的性能指标。
- **自底向上**：从 CPU 统计数据开始，上升到应用结构。更关注 CPU 使用。

## 改善性能三方面

1. **性能监控**：非侵入式收集运行性能数据
2. **性能分析**：侵入式收集性能数据（可能影响吞吐量）
3. **性能调优**：为改善响应或吞吐量修改参数、源代码或配置

## Linux 性能监控命令

| 监控项 | 命令 |
| --- | --- |
| CPU 使用率 | vmstat, mpstat, top |
| CPU 调度队列 | vmstat |
| 内存利用率 | free, vmstat |
| 网络 I/O | nicstat |
| 磁盘 I/O | iostat |

## HotSpot VM 三个组件

- VM 运行时
- JIT 编译器
- 内存管理器
